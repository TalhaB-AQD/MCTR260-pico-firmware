Detailed Description

Mecanum drive motion profile: the command pipeline.

FULL DATA PIPELINE (one joystick update, ~20ms):

  1. Flutter app sends JSON: {"type":"control","right":{"x":50,"y":80},...}
  2. command_parser extracts vx=50, vy=80, omega=0 from the joystick
  3. This module calls mecanum_calculate(vx, vy, omega) -> 4 wheel speeds
  4. Speeds are SCALED to motor units:
    • DC motors: +/-100 -> +/-255 PWM (multiply by 2.55)
    • Steppers: +/-100 -> +/-STEPPER_MAX_SPEED steps/sec
  5. For steppers: speeds are written to g_targetSpeeds[] via MUTEX Core 1 reads these speeds and generates step pulses

MULTICORE HANDOFF (stepper path only): Core 0 uses mutex_try_enter(), which is NON-BLOCKING. If Core 1 holds the mutex (it's mid-I2C write), Core 0 skips this update. At 50Hz command rate, dropping one update is imperceptible. The alternative (mutex_enter_blocking) would stall Core 0 and delay BLE processing.

LEFT/RIGHT INPUT MAPPING: The Flutter app's "left" and "right" controls map differently per vehicle: mecanum: left=dial(rotation) right=joystick(translation) fourwheel: left=dial(steering) right=slider(throttle) tracked: left=slider(L) right=slider(R)

void profile_mecanum_apply ( const control_command_t * cmd)

Apply the mecanum drive profile to a control command.

INPUT MAPPING: The Flutter app sends two control inputs: "left" and "right". For the mecanum profile: Right (joystick): x -> vx (strafe), y -> vy (forward) Left (dial): value -> omega (rotation) If left is a joystick instead of a dial, its X axis is used for rotation.

MOTOR TYPE BRANCHING: DC motors: speed (-100..+100) multiplied by 2.55 to fill the 0-255 PWM range. motor_set_pwm() is called directly on Core 0 (no cross-core handoff needed). Stepper motors: speed (-100..+100) scaled to steps/sec by STEPPER_MAX_SPEED/100. Written to g_targetSpeeds[] behind a mutex for Core 1 to pick up.

NON-BLOCKING MUTEX: mutex_try_enter() returns false if Core 1 holds the mutex (it's mid-I2C write). In that case, this command is silently dropped. At 50Hz command rate, one dropped command is imperceptible.
