Detailed Description

JSON command parsing implementation.

PROTOCOL: The Flutter app sends JSON commands over BLE with this structure: {"type":"control", "vehicle":"mecanum", "left":{...}, "right":{...}, ...} {"type":"heartbeat"}

This parser uses ArduinoJson (v7) with a JsonDocument that auto-allocates on the stack. The parsed result is stored in a control_command_t struct (see command_packet.h for the exact field layout and example JSON).

The parser is stateful: it keeps the last valid command in s_currentCommand. This allows other modules to call command_get_current() at any time to read the most recent command, useful for telemetry reporting and safety logic.

ESP32 PARITY: The JSON format and struct layout are identical to the ESP32 firmware. Both firmwares can be controlled by the same Flutter app without changes.

bool command_parse ( const char * json,
control_command_t * cmd )

Parse a JSON command string into a control_command_t struct.

Uses ArduinoJson v7's JsonDocument which auto-sizes on the stack. Typical command JSON is ~200 bytes, well within stack limits.

The parser handles two command types:

  • "heartbeat": no payload, just resets the safety watchdog.
  • "control": full joystick/speed/aux/toggle payload.

The parsed command is COPIED into s_currentCommand so other modules can read the latest command at any time via command_get_current().
