Detailed Description

DC motor configuration (one per physical motor).

Each motor gets its own config struct with pin numbers, driver type, and direction. These are defined in project_config.h and passed to the MotorDC constructor during motors_init().

See also
project_config.h where these values are set for your robot

Member Data Documentation

◆ direction

int8_t MotorDCConfig::direction

Direction multiplier: 1 for normal, -1 to reverse. Motors on opposite sides of a robot spin in opposite physical directions for the same "forward" command. Use -1 to correct this in software instead of rewiring.

◆ driverType

◆ enabled

bool MotorDCConfig::enabled

Master on/off switch. Set false to disable this motor completely (useful for testing with fewer motors).

◆ index

uint8_t MotorDCConfig::index

Motor index (0-3 for a 4-wheel mecanum robot). 0=Front-Left, 1=Front-Right, 2=Back-Left, 3=Back-Right

◆ pinA

int8_t MotorDCConfig::pinA

First GPIO pin. For DRV8871/DRV8833: PWM output to IN1. For L298N: direction pin IN1.

◆ pinB

int8_t MotorDCConfig::pinB

Second GPIO pin. For DRV8871/DRV8833: PWM output to IN2. For L298N: direction pin IN2.

◆ pinEnable

int8_t MotorDCConfig::pinEnable

Enable/speed GPIO pin (L298N only). PWM output that controls motor speed. Set to -1 for DRV8871/DRV8833 (they don't have a separate enable).
