Detailed Description

Abstract base class for all motor types (DC, stepper, servo).

WHAT IS AN ABSTRACT BASE CLASS? This file defines a "template" that all motor types must follow. It lists the functions every motor must have (init, stop, update) but doesn't implement them. Each specific motor type (DC or stepper) provides its own implementation. This is C++ polymorphism: the rest of the firmware calls motor->stop() without needing to know whether it's a DC motor or a stepper underneath.

CLASS HIERARCHY: MotorBase (this abstract class) +– MotorDC: PWM control via H-bridge (DRV8871, DRV8833, L298N) +– MotorStepper: Step/dir control via MCP23017 (TMC2209, A4988, DRV8825)

WHY setTarget() IS NOT IN THE BASE CLASS: DC motors take a PWM value (-255 to +255). Steppers take a speed in steps/sec (float). Servos would take an angle (0-180 degrees). Forcing these into one function signature would require ugly workarounds. Instead, motor_manager.cpp casts to the specific subclass when needed.
